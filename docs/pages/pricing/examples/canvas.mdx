This example estimates a monthly bill for a multiplayer canvas with
[Presence](/docs/ready-made-features/presence). It assumes <strong>2,000 monthly
active</strong> users, with <strong>400 collaborating daily</strong> for
<strong>80 minutes each</strong> (22 working days).

Recommended plan: [Team](/docs/pricing/plans/team) at $900/mo ($1,000 credits
included).

## What you’re billed for

[Realtime collaboration minutes](/docs/pricing/plans/team#Metered-usage): active
time each user spends connected in a room, billed at $0.002 per minute.

## Usage estimate

| Item                                                                     | Calculation                  | Total        |
| ------------------------------------------------------------------------ | ---------------------------- | ------------ |
| [Realtime collaboration minutes](/docs/pricing/plans/team#Metered-usage) | 400 daily × 80 min × 22 days | 704,000 mins |
| Session cost                                                             | 704,000 × $0.002             | $1,408       |
| **Total usage**                                                          |                              | **$1,408**   |

## Your bill

| Item                        | Billed monthly | Billed annually |
| --------------------------- | -------------- | --------------- |
| Team plan ($1,000 credits)  | $900/mo        | $750/mo         |
| Usage beyond credits (PAYG) | $408           | $408            |
| **Total**                   | **$1,308/mo**  | **$1,158/mo**   |

Credits apply to usage first; overage is billed at the same rates. See
[Billing](/docs/pricing/billing) for how credits, plan base, and PAYG add up.
Billing annually saves 2 months on the plan price; overage is unchanged.

The $900/mo Team plan includes $1,000 in credits, $100 more than the plan price.
You only pay overage above that allowance.
